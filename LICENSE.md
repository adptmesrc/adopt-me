## 🔧 Common Issues & Fixes

### ❌ "GUI doesn't appear after executing"
**Fix:** Make sure your executor is fully attached to Roblox *before* executing. The status indicator in your executor should show green/attached. Re-attach and try again. If the issue persists, try a fresh Roblox relaunch.

### ❌ "Auto Farm Money stops after a few minutes"
**Fix:** This is usually caused by an NPC task getting stuck. Enable the **Auto Retry** toggle in the GUI's Farm Settings panel. If the problem continues, rejoin the server and re-execute — some servers have anti-farm detection that kicks idle scripts.

### ❌ "Pet Spawner shows pets but they disappear quickly"
**Fix:** The visual pet spawner is client-side only. Pets are visible to you but not other players, and they reset on server re-renders. This is expected behavior — the feature is designed for personal showcase, not permanent server-wide display.

### ❌ "Auto Accept is approving trades I don't want"
**Fix:** Open the GUI's Trade Filter settings and configure your minimum rarity threshold. Set it to **Legendary** or higher to prevent the script from auto-accepting common/uncommon pet offers.

### ❌ "Script crashes on mobile (Android)"
**Fix:** Ensure your Android version is 8.0 or higher and that your mobile executor is up to date. Close background apps to free RAM before executing — running Roblox and the executor simultaneously requires at least 3 GB of free RAM.

### ❌ "Infinite Bucks module isn't working"
**Fix:** This module requires a Level 8 executor. If you're using a lower-level executor, the Bucks injection will be blocked. Switch to a Level 8-compatible executor such as Volcano or Synapse X. Also verify you're on the latest script version via the **adopt me script discord server**.

### ❌ "Script was working yesterday but broke today"
**Fix:** Adopt Me updates frequently. When the game patches, scripts often need updates too. Check the Changelog above for the latest version. Join the official Discord for immediate patch notifications.
